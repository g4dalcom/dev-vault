## 📝 문제

두 정수 `X`, `Y`의 임의의 자리에서 공통으로 나타나는 정수 k(0 ≤ k ≤ 9)들을 이용하여 만들 수 있는 가장 큰 정수를 두 수의 짝꿍이라 합니다(단, 공통으로 나타나는 정수 중 서로 짝지을 수 있는 숫자만 사용합니다). `X`, `Y`의 짝꿍이 존재하지 않으면, 짝꿍은 -1입니다. `X`, `Y`의 짝꿍이 0으로만 구성되어 있다면, 짝꿍은 0입니다.

예를 들어, `X` = 3403이고 `Y` = 13203이라면, `X`와 `Y`의 짝꿍은 `X`와 `Y`에서 공통으로 나타나는 3, 0, 3으로 만들 수 있는 가장 큰 정수인 330입니다. 다른 예시로 `X` = 5525이고 `Y` = 1255이면 `X`와 `Y`의 짝꿍은 `X`와 `Y`에서 공통으로 나타나는 2, 5, 5로 만들 수 있는 가장 큰 정수인 552입니다(`X`에는 5가 3개, `Y`에는 5가 2개 나타나므로 남는 5 한 개는 짝 지을 수 없습니다.)  
두 정수 `X`, `Y`가 주어졌을 때, `X`, `Y`의 짝꿍을 return하는 solution 함수를 완성해주세요.

##### 제한사항

-   3 ≤ `X`, `Y`의 길이(자릿수) ≤ 3,000,000입니다.
-   `X`, `Y`는 0으로 시작하지 않습니다.
-   `X`, `Y`의 짝꿍은 상당히 큰 정수일 수 있으므로, 문자열로 반환합니다.

---

##### 입출력 예

| X       | Y        | result |
|:------- |:-------- |:------ |
| "100"   | "2345"   | "-1"   |
| "100"   | "203045" | "0"    |
| "100"   | "123450" | "10"   |
| "12321" | "42531"  | "321"  |
| "5525"  | "1255"   | "552"       |


---

##### 입출력 예 설명

**입출력 예 #1**

-   `X`, `Y`의 짝꿍은 존재하지 않습니다. 따라서 "-1"을 return합니다.

**입출력 예 #2**

-   `X`, `Y`의 공통된 숫자는 0으로만 구성되어 있기 때문에, 두 수의 짝꿍은 정수 0입니다. 따라서 "0"을 return합니다.

**입출력 예 #3**

-   `X`, `Y`의 짝꿍은 10이므로, "10"을 return합니다.

**입출력 예 #4**

-   `X`, `Y`의 짝꿍은 321입니다. 따라서 "321"을 return합니다.

**입출력 예 #5**

-   지문에 설명된 예시와 같습니다.

---

### 🔍 정답

```java
class Solution {
    public String solution(String X, String Y) {
        StringBuilder sb = new StringBuilder();
        
        int[] checkX = new int[10];
        int[] checkY = new int[10];
        
        for (String x : X.split("")) {
            checkX[Integer.parseInt(x)]++;
        }
        
        for (String y : Y.split("")) {
            checkY[Integer.parseInt(y)]++;
        }
        
        for (int i = 9; i >= 0; i--) {
            while (checkX[i] > 0 && checkY[i] > 0) {
                checkX[i]--;
                checkY[i]--;
                sb.append(i);
            }
        }
        
        if (sb.toString().equals("")) return "-1";
        else if (sb.toString().startsWith("0")) return "0";
        else return sb.toString();
    }
}
```
- 생각보다 너무너무 어려웠다.
- 처음에는 String 을 char[] 로 형변환을 하고 char[] 배열로 비교를 해서 다시 String에 붙이는 식으로 했는데 테스트에서 시간초과가 나는 경우가 생겼다. 찾아보니 StringBuilder를 이용해야 한다는 것이었는데 이리저리 해봐도 답이 나오질 않아서 답안을 찾아보았던 문제이다 ㅠㅠ
- 0~9까지의 배열을 두 개 만들어서 해당 숫자가 포함되면 같은 숫자의 인덱스를 +해서 초기화를 하고 
- 인덱스를 거꾸로 돌면서(높은 숫자를 출력해야 하므로 높은 숫자부터 StringBuilder에 붙이기 위함) 두 배열이 모두 0보다 크면 서로 짝꿍인 것이므로 StringBuilder에 추가해주는 식이다.
- 또한 모든 짝꿍수가 "0" 이면 0을 리턴해야 하는 것도 고민이 되었었는데, 높은 숫자부터 StringBuilder에 붙이므로 모든 짝꿍수가 0으로만 이루어져있다면 startsWith로 걸러낼 수가 있다는 것도 신박했다.
- 풀어보면서 형변환에 대해 좀 더 공부해야겠다는 생각이 들었다 ㅠㅠ
